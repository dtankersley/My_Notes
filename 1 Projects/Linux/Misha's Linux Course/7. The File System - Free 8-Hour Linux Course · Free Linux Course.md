## Linux File System Structure
- The [[Linux]] file system is a hierarchical structure where everything is organized in a tree-like format, with the root directory, denoted by "/", serving as the top of the file system, and all other directories and files branching down from it.
- In Linux, everything is represented as a file, including regular files, directories, devices, network sockets, and processes, which allows for interaction with almost anything using the same tools, following the core [[Unix]] philosophy.
- Key directories in the Linux file system include /home for user home directories, /etc for system configuration files, /var for variable data, /tmp for temporary files, /usr for user programs and data, /bin for essential command binaries, /sbin for system administration binaries, /opt for optional/third-party software, /dev for device files, and /proc for process and kernel information.
- Absolute paths in Linux start from the root directory, denoted by "/", and always work regardless of the current location, whereas relative paths start from the current location, and special shortcuts include "." for the current directory, ".." for the parent directory, and "~" for the user's home directory.

## Navigation Commands
- Essential navigation commands in [[Linux]] include "pwd" to print the working directory, "cd" to change the directory, "cd ~" or "cd" to go to the home directory, "cd .." to go up one level, and "cd -" to go to the previous directory.
- The "ls" command is used to list directory contents, with options such as "-l" for long format, "-a" for showing all files including hidden ones, and "-h" for human-readable file sizes, and the "tree" command is used to visualize the directory structure.
- Working with files and directories in Linux involves using commands such as "mkdir" to create a directory, "touch" to create an empty file, "cp" to copy files, and "rm" to delete files, with various options available to modify the behavior of these commands.

## File System Operations Overview
- Understanding the Linux file system and being able to navigate and work with files and directories is fundamental to using Linux effectively, and is a crucial skill for anyone looking to work with Linux, including those pursuing DevOps roles.
- The section of the [[Linux]] course covers various file system operations, including copying, moving, renaming, and deleting files and directories, with commands such as `cp`, `mv`, and `rm`, and flags like `-r` for recursive operations and `-i` for interactive prompts.

## File Manipulation Commands
- The `mv` command can be used to move or rename files, and the `rm` command can be used to delete files, with the `-r` flag allowing for recursive deletion of directories and their contents, but the `rm -rf /` command is extremely dangerous and should never be used.
- The `find` command can be used to search for files by name, type, or size, with options such as `-name` for searching by file name, `-type` for searching by file type, and `-size` for searching by file size.

## File Search and Command Analysis
- The `locate` command is a faster alternative to `find`, but it uses a database that may be outdated, and can be updated with the `sudo updatedb` command, and the `which` and `command -v` commands can be used to find the location of a command's executable.
- The `type` command can be used to determine the type of a command, whether it is an alias, a builtin, or an external command, and the `man` command can be used to access the official documentation for a command, with options such as `-k` for searching man page descriptions.
- The `help` command can be used to access help for [[Bash (Unix shell) | shell]] builtins, and the `--help` flag can be used to access a quick reference for most commands, with the `man` command providing more detailed information, and the course emphasizes the importance of being careful when using commands like `rm -rf` and using the correct commands and options to achieve the desired results.

## File System Structure Recap
- The [[Linux]] file system has a single directory tree that starts at the root directory, represented by "/", and key directories include /home for user files, /etc for configuration files, and /var for log files.
- To navigate the file system, users can utilize the pwd command to see their current location, the cd command to move to a different directory, and the ls command to list the files and directories in their current location.

## Core File Operations
- Various commands are available for file operations, including mkdir for creating directories, touch for creating empty files, cp for copying files, mv for renaming or moving files, and rm for deleting files.
- The find command is used to search the file system, while the locate command uses a database to find files, and the type command is more informative than the which command for understanding commands.

## Documentation and Help Systems
- For documentation and help, users can use the man command for full documentation of a command, the help command for [[Bash (Unix shell) | shell]] builtins like cd and type, and the --help option for a quick reminder of command options.
- The concept of paths is important in the [[Linux]] file system, with absolute paths starting from the root directory, relative paths being relative to the current location, and the working directory being the current location.

## Advanced Concepts and Shell Builtins
- Other important concepts include hidden files, which are files whose names start with a dot and are not shown by default in the ls command, and recursive operations, which operate on a directory and all its contents, including subdirectories.
- The home directory, usually represented as ~/ or /home/username, is the user's personal directory, and the man page is the primary documentation source for [[Unix]]/Linux systems, providing detailed information about commands and their usage.
- [[Bash (Unix shell) | Shell]] builtins, such as cd, type, and help, are commands that are built directly into the shell, rather than being external programs, and can be used to perform various tasks and operations in the Linux file system.
