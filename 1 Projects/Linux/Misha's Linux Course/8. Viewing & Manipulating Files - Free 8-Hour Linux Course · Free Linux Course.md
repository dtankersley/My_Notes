## Introduction to Linux File Commands
- The foundation of [[Linux]] is crucial for building containers, Kubernetes, CI/CD, and cloud systems, and understanding how to view and manipulate files is essential for reading logs, configuration files, and comprehending what is on a system.
- The goal is to be able to view [[File (command) | file]] contents, search for text, and understand file information without having to open an editor, and this can be achieved by using various Linux commands such as [[Cat (Unix) | cat]], [[Less (Unix) | less]], head, [[Tail (Unix) | tail]], and [[Grep | grep]].

## Viewing and Searching File Contents
- The cat command is used to display the entire contents of a file, but it can be overwhelming for large files, whereas the less command allows users to page through files and scroll through the content.
- The head and tail commands are used to view the first and last lines of a file, respectively, and the tail -f command is used to watch files in real-time, which is essential for debugging and monitoring log files.
- The grep command is a powerful tool for searching for patterns in files, and it supports various options such as case-insensitive search, line numbers, and recursive search, making it a versatile command for finding specific text within files.

## File Type Identification and Importance
- The [[File (command) | file command]] is used to determine the type of a file, and it examines the content of the file to determine its type, rather than relying on file extensions, which can be useful for identifying the type of a file.
- Understanding how to use these commands is crucial for working with files in [[Linux]], and it is an essential skill for anyone working in DevOps or related fields, as it enables them to efficiently manage and troubleshoot systems.

## File Information Commands
- The `stat` command is used to display detailed information about a file, including its size, permissions, owner, timestamps, and more, as shown in the example `stat /etc/passwd`.
- The `wc` command is used to count the number of lines, words, and characters in a file, and can be used with various options, such as `wc -l` to count lines, `wc -w` to count words, and `wc -c` to count characters.

## Disk Usage and Space Commands
- The `du` command is used to estimate the disk usage of a [[File (command) | file]] or directory, and can be used with options such as `-s` to summarize and `-h` to make the output human-readable, as shown in the example `du -sh /var/log`.
- The `df` command is used to display information about the disk space usage of mounted filesystems, and can be used with options such as `-h` to make the output human-readable, as shown in the example `df -h`.

## Investigating Directories and Files
- To investigate the `/etc` directory, commands such as `ls /etc | wc -l` can be used to count the number of files and directories, and `[grep](1ec0b614-0186-4fac-86b5-222d134349c1) -r "network" /etc/` can be used to find files containing the word "network".
- To find the largest file in `/var/log`, commands such as `ls -lhS /var/log | head -5` or `du -h /var/log/* 2>/dev/null | sort -h | [tail](26545362-7f3b-4fc3-83f4-034308905f0c) -5` can be used.

## Checking System Disk Space
- To verify the total disk space used on the system, the command `df -h /` can be used.

## Summary of Essential Commands
- Various commands, including `cat`, `less`, `head`, `tail`, [[Grep | grep]], `file`, `stat`, `wc`, `du`, and `df`, are essential for viewing and manipulating files in [[Linux]], and are used for tasks such as displaying [[File (command) | file]] contents, searching for patterns, and estimating disk usage.
