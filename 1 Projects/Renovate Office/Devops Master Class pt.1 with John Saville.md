%% Waterfalls and Being Agile %%

* Process and the project manager is a big piece of that continuous cycle
* Waterfall used to be very popular as large, dependent phases*


Scrum:
working from backlog of requests
backlog is handled by priority in sprints (1-4 weeks)
each backlog item has a point value
Each backlog can handle a rough amount of points (10 points?) (capacity)
Backlog items have varying point values
daily standup meetings - discuss priorities of the day


 * Today, more organizations use the AGILE mindset
 * There are different frameworks of AGILE
KANBAN:
Continuous strem of work
work in Progress (Wip) limit  
build - test - deploy - each 'phase' cn handle so many units - pull from backlog as space allows
* Start how yu work - create incremental change - respect roles - leadership all - visualize board - limit WIP - always pull - use good definitions of the steps & stages - 


KANBAN Pizza

a pizza order is a BANBAN - it starts with a ticket (requirements) from the customer/stakeholder
the ticket is added to the backlog
each person in the kitchen can handle X-amount of work at a time - when work is completed, they can pull new work from the backlog - 
The process moves from assemply to the oven to cutting/prep, boxing and delivery (table/pickup/delivery)
**Each stage has a WIP limit**
