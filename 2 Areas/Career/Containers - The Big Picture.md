
1. What is a container?
   A container is a self-contained application that runs in its own environment, fully independant of any other soort of Host systems. Think of it like a virtualized server that runs ONLY what the application needs in order to run successfully.The container runs in a minimal Linux OS host, with only the exact dependancies and SW libraries that it needs. These dependancies and librarys will then not conflict with any other host software whereever it should run. 
    
2. Why companies use containers instead of "traditional" deplyments:
   with containers, the application runs ONLY from its own  envroinment and cannot be influenced or corrupted by any other system conflicts that could be introduced . The environment is self-contained and 100% controlled, so that developers need not worry about external hardware or software conflicts.
3. Why you need to learn containers to earn over $100k:
    Companies will be switching to containerized environments to reduce support issues and increase reliability.
